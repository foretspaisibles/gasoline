Module type Gasoline_Generic_Definition.VALUE

module type VALUE = sig .. end
Description of application values.

The preparation, or formatting, of these values is governed by two sets of parameters. The first set of parameters is referred to collectively as a locale. It can be used to actually emulate locale settings as they are provided by other systems. The second set of parameters is referred to as a format, it is much like formatting information used by the printf facility.

The two sets of parameters locale and format are distinct and orthogonal: the first one is usually bound to an output session while the second is set for each value.

type t
The type of values that can be conveyed by application messages and application configuration parameters.
type locale
The type of localisation parameters.
type format
The type of formatting informations for values.
val stdformat : format
The standard format.
val format_to_string : format -> string
Convert format information to a concrete form.
val format_of_string : string -> format
Convert concrete form to format information.

It returns stdformat when called on the empty string. The details of the format syntax is implementation-dependant.
Raises Invalid_arg when the given argument cannot be converted.

val add_formatted : buffer ->

       buffer ->
       format ->
       t -> unit
Write a formatted value to a data buffer.

The first buffer given in argument is a scratch buffer that is empty when the function is called and can be used as a scratch area. The function is expected to add the appropriate words to the second buffer.
