Formatted output

The low-level functions add, add_foramtted and add_text should only be used in emergency cases. The preferred way to output values is through the use of messages.

type locale = CApplication.Locale.t
The type of localisation parameters.
type format = {
   specifier : char;
   left_justify : bool; (*Left-justify within the given field width; Right justification is the default (see width sub-specifier).*)
   always_signed : bool; (*Forces to precede the result with a plus or minus sign (+ or -) even for positive numbers. By default, only negative numbers are preceded with a - sign.*)
   align_sign : bool; (*If no sign is going to be written, a blank space is inserted before the value.

This implies that the align_sign flag is ignored when the the always_signed flag is set, as clarified by ANSI. 

    ``If the space and + flags both appear, the space
      flag will be ignored.''
      -- ANSI X3J11
*)
   alternative_format : bool; (*Choose an alternative format, depending on the value.*)
   pad_with_zeroes : bool;
   width : int;
   precision : int;
}

type buffer = Buffer.t
The type of data buffers.
val add_formatted : buffer ->
       locale ->
       buffer ->
       format -> t -> unit
Write a formatted value to a data buffer.

The first buffer given in argument is a scratch buffer that is empty when the function is called. The function is expected to add the appropriate words to the second buffer.
