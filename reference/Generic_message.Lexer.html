Lexing message strings containing variables

A message string containing variables is encoded as a string. Within this string, a sequence ${identifier} indicates a variable, the identifier can be any non empty sequence of characters from the set [A-Za-z0-9_]. A sequence ${identifier:format} indicates a variable with formatting information (pretty much like printf). Formatting information can be any sequence of characters from the set [A-Za-z0-9_+-.,] but this string must also be validated by the function V.format_of_string.

An escaped $ character is a $ that immediately follows a backslash character; it then stands for a plain $.
